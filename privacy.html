Privacy Policy

Service: Personal Discord SMS Notification Bot

Operator: Jon Wetzler

Contact: jonjayj@gmail.com

Last updated: May 12, 2026

1. Overview

This service is an automated application-to-person (A2P) notification system operated by a single individual for their own personal use. There are no external users, customers, or third-party recipients. Messages are sent exclusively to the operator's own verified mobile number. This service is not commercial and is not offered to the public.

2. Information Collected

The application reads message content from channels in a private Discord server that the operator owns and administers. Specifically, the application reads:

  • Message text
  • Channel name and ID
  • Author display name
  • Attachment count (presence only, not contents)

No data from any other Discord server or any other user is collected.

3. How Information Is Used

The collected information is used solely to construct an SMS notification that is sent to the operator's own mobile number, alerting them of activity in their own Discord server. The data is:

  • Not persistently stored beyond transient in-memory processing during the API call
  • Not sold, shared, analyzed, or used for marketing
  • Not shared with any third party other than the SMS carrier (Twilio) required to deliver the message

4. Data Sharing

Message data flows only between the following systems, all operated by or on behalf of the operator:

  • Discord API (source of the message)
  • The application running on Railway cloud infrastructure (the operator's deployment)
  • Twilio API (the SMS carrier delivering the notification)
  • The operator's own mobile device

Twilio's handling of message data is governed by Twilio's privacy policy.

5. Recipients

The application is hardcoded to send to a single configured phone number, which belongs to the operator and has been verified with Twilio as a Verified Caller ID. No other recipient can receive messages without the operator redeploying the application with a different configured number.

6. Consent and Opt-Out

Consent to receive messages is established by the operator configuring their own phone number into the application's environment variable SMS_TO_NUMBER at deployment time. The operator can stop receiving messages at any time by:

  • Replying STOP to any message (handled by Twilio's standard STOP filter)
  • Removing or changing the SMS_TO_NUMBER environment variable in the application's hosting platform
  • Stopping or deleting the application deployment

The operator may reply HELP for assistance per Twilio defaults.

7. Data Retention

The application does not retain message data. Hosting platform logs may briefly capture diagnostic information for operational debugging, retained per Railway's standard log retention policy.

8. Contact

Questions about this policy can be directed to jonjayj@gmail.com.
